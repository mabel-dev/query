Query Editor

Query supports a limited Data Query Language.

SELECT * | select_expression | DISTINCT FROM dataset [WHERE filter_expression] [GROUP BY column_name] [ORDER BY column_name ASC|DESC] [LIMIT n]

JOINs are not supported implemented

select_expression

Sets the columns to populate the result set with, in a comma separated list. select_expressions can contain functions to perform on columns in the source dataset. At least one expression is required, GROUP BY statements must contain aggregation functions to perform on the groups.

Filter Operations:

  • equals (=), also as '=='
  • not equals (<>), also as '!='
  • greater than (>)
  • greater than or equal (>=)
  • less than (<)
  • less than or equal (<=)
  • is (IS)
  • isn't (IS NOT)
  • SQL like (LIKE)
  • not SQL like (NOT LIKE)

Aggregation Functions:

  • COUNT
  • SUM
  • MAX
  • MIN

Scalar Functions:

  • DAY - extract the day of the month from a date
  • MONTH - extract the month of the year from a date
  • YEAR - extract the year from a date
  • DATE - extract/convert the date from a timestamp
  • QUARTER - extract the quarter of the year from a date
  • UCASE - convert a string to upper case
  • LCASE- convert a string to lower case
  • LEN - calculate the length of a string
  • ROUND - round a number
  • TRUNC - truncate a number
  • INT - convert a number or string to an integer (same logic as INT)
  • FLOAT - convert a number or string to a floating point number
  • BOOLEAN - convert a string TRUE/FALSE to a boolean
  • ISNONE - returns TRUE (boolean) if a value is none/null
